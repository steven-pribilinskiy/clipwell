The picker is a pre‑warmed window that appears the instant you press the hotkey.
History loads a page at a time and **fetches older items as you scroll**, so even a
very long history stays fast (the list is virtualized).

## Find and paste something you copied earlier

1. Press **Alt+Shift+V**. The picker appears centered, search box focused.
2. Start typing to filter by content.
3. Press **Enter** (or double‑click) to copy the item — Clipwell restores focus to
   the app you were in and pastes it there.
4. Press **Escape** (or click away) to dismiss without pasting.

> Clipwell remembers which window was focused when you opened the picker, so the
> paste lands back in the right place.

## Recognize items at a glance

Every item is classified and shown with a glyph:

| Glyph | Kind  | Example |
|-------|-------|---------|
| 🔗    | url   | `https://avaloniaui.net` |
| 🔀    | github-pr | `https://github.com/org/repo/pull/123` |
| 🎫    | jira-issue | `PROJ-1234` |
| ✉     | email | `you@example.com` |
| 🎨    | color | `#3366ff` |
| 📁    | path  | `C:\Users\you\file.txt` |
| `{ }` | code  | `const sum = (a, b) => a + b;` |
| 🖼     | image | a screenshot |
| 📄    | text  | anything else |

Classification is done by pluggable detectors, so the set of kinds can be extended.
**Image items show a thumbnail**, and **URLs show the site's favicon** (fetched
directly over HTTPS — no third‑party favicon service) in place of the glyph.
Each item also shows **which app it was copied from** ("Tabby · just now"),
captured at copy time.

## Compact and Detail views

Toggle between **Compact** (a dense single‑column list) and **Detail** (list on the
left, a full preview pane on the right) with the **▦ Detail / ▤ Compact** button.
The preview pane shows the selected item's kind, source, and full content — the
complete text, or a full‑resolution image.

## Peek without switching: hover & Quick Look

Hover any item to see its full content in a tooltip. For a bigger look, select an
item and press **Ctrl+Y** for **Quick Look** — a full‑window overlay with the
complete text (scrollable) or a full‑resolution image. Press **Ctrl+Y** again,
**Escape**, or click the backdrop to close.

## Filter to what you need

The tabs below the search box narrow the list instantly:

- **All** — everything, newest first (pinned items float to the top).
- **📌 Pinned** — only items you've pinned for reuse.
- **🔒 Sensitive** — only items you've flagged as sensitive.

The **type dropdown** filters by detected kind, and the **grouping dropdown** can
group the list **by date** (Today / Yesterday / Previous 7 days / …) or **by source
app**, with a header above each group.

Keyboard shortcuts work on the selected item: **Ctrl+1 / Ctrl+2 / Ctrl+3** switch
tabs, **Delete** removes an item, **Ctrl+P** toggles pin, **Ctrl+E** toggles the
sensitive flag, **F2** renames (alias), and **Ctrl+Y** opens Quick Look.

## Pin snippets you reuse

Pin the things you paste over and over — a sign‑off, a command, a code snippet.
Select an item and press **Ctrl+P** (or use the **📌 Pinned** tab to manage them).
Pinned items are sorted to the top of the list and survive history retention sweeps.

## Hide sensitive items

Flag an item as **sensitive** (**Ctrl+E**) and its preview is masked in the list as
`•••••••••••`, so a password or token doesn't sit in plain sight while you scroll.
Sensitive image items don't render a thumbnail.

## Settings

Open **Settings…** from the tray menu to tune:

- **Appearance** — System (default), Light, or Dark theme.
- **Default view** — Compact or Detail.
- **Default grouping** — None, by date, or by source.
- **Row metadata** — show/hide the source app and time on each row.
- **History retention** — 7 / 30 / 90 days, or **Forever**.
- **Picker position** — open at the mouse cursor or centered.
- **Global hotkey** — rebindable: click **Change…** and press a new combo (default
  Alt+Shift+V). Applies live on Windows.

## Check performance & daemon health

Open **Diagnostics…** from the tray menu to see the daemon status (and its database
path) and the picker's **show-cycle timings** from `perf.log` — count, median, min,
and max — so you can confirm the picker stays in its single‑digit / one‑frame range.
Clear the log or refresh from the same window.

## Keep it running in the background

Clipwell lives in the system tray. Closing the picker window doesn't quit it — use
the tray menu's **Quit** for that. Only one instance runs at a time.
