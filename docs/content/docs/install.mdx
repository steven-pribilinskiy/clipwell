---
title: Install
description: Build and run Clipwell on Windows, macOS, and Linux.
---

Clipwell needs the [.NET 10 SDK](https://dotnet.microsoft.com/download). The daemon
runs everywhere; the picker UI is [Avalonia](https://avaloniaui.net) (also
cross‑platform).

## Build

```sh
git clone https://github.com/steven-pribilinskiy/clipwell
cd clipwell
dotnet build
```

## Run

```sh
dotnet run --project daemon   # http://127.0.0.1:8787 (override with CLIPWELL_URL)
dotnet run --project ui       # the picker
```

## Platform notes

### Windows
Full support: native clipboard watcher (text, HTML, images), global hotkey
(**Alt+Shift+V** — Win+V is reserved by the OS), tray icon, and paste‑into‑the‑
source‑app.

### macOS / Linux
The daemon watches the clipboard by polling the platform CLI, so install one:

- **macOS** — `pbpaste` ships with the OS.
- **Linux (Wayland)** — `wl-clipboard` (`wl-paste`).
- **Linux (X11)** — `xclip`.

The tray icon is cross‑platform (Avalonia). The **global hotkey** and
**paste‑into‑the‑source‑app** are now implemented natively per OS:

The hotkey is **rebindable** in Settings → Global hotkey (default below per OS;
applies live on Windows, on next launch on macOS/Linux).

- **macOS** — hotkey is **Option+Shift+V** (Carbon `RegisterEventHotKey`); pasting
  synthesizes Cmd+V and needs **Accessibility** permission (System Settings →
  Privacy & Security → Accessibility). Without it, the item is still copied.
- **Linux (X11)** — hotkey is **Alt+Shift+V** (`XGrabKey`); pasting uses `xdotool`
  (install it for paste‑back). Under a pure **Wayland** session there's no global
  key‑grab API, so the hotkey falls back to tray‑only.

These native paths are exercised by CI's cross‑platform build; the GUI behavior is
still being validated on real macOS/Linux hardware. If a hotkey can't register, the
app keeps working from the tray.

## Data location

History lives in a SQLite database under your OS app‑data folder
(`%APPDATA%\Roaming\Clipwell` on Windows, `~/.config/Clipwell` on Linux,
`~/Library/Application Support/Clipwell` on macOS). Override with `CLIPWELL_DATA_DIR`.
